# scripts

Developer and CI helper scripts.

| Script                | Purpose                                             |
|-----------------------|-----------------------------------------------------|
| `build-all.sh`        | Build and test every service via the Gradle wrapper.|
| `run-service.sh`      | Run a single service locally (`run-service.sh auth-service`). |
| `backup-databases.sh` | Dump every service DB (`pg_dump` via `docker exec`); optional AES-256 + offsite upload. |
| `restore-database.sh` | Restore ONE service DB from a dump (destructive; guarded by a confirmation prompt). |
| `verify-backup.sh`    | Prove a dump restores cleanly into a disposable temp DB (live data untouched). |
| `restore-drill.sh`    | End-to-end restore drill: seed canary → backup → restore → assert data **and** parking's PostGIS objects survive. Runs in CI (`backup-restore-drill.yml`). |
| `seed-real-e2e.sh`    | Idempotently seed the real-stack E2E accounts (USER/MODERATOR/ADMIN, ACTIVE + email-verified, exact roles) via `docker exec psql`. BCrypt hashing happens in-DB (pgcrypto); passwords are never printed or passed on a command line. |
| `cleanup-real-e2e.sh` | Remove data generated by a real-stack E2E run (spots, media, reports, notifications), scoped strictly to the seeded accounts / test address marker; `--accounts` also drops the seed accounts. Supports `--dry-run`. |
| `outbox-deadletter-recovery.sh` | Private operator tool to inspect, retry, bulk retry, or acknowledge outbox dead-letter rows with durable DB audit rows. |
| `kafka-dlt-redrive.sh` | Private operator wrapper for the Java Kafka DLT redrive tool. Dry-run by default; preserves Kafka headers and caps redrive batches. |
| `chaos-compose-validation.sh` | Guarded local/CI Docker Compose chaos drill: stop/restart Kafka, Redis, representative Postgres, MinIO, gateway, notification-service, and analytics-service, then verify recovery to healthy. |
| `../benchmarks/scripts/collect-prometheus-baseline.sh` | Snapshot Prometheus HTTP, JVM, Hikari, Kafka and outbox metrics during a performance run. |

All scripts assume they are run from the repository root or use paths relative
to it.

The backup/restore/drill scripts require the `parkio-postgres-*` containers to be
running and load DB credentials from `PARKIO_ENV_FILE` (e.g. `docker/.env`). See
[`docker/README.md`](../docker/README.md) §"Backups & restore" for the full runbook.

`seed-real-e2e.sh` / `cleanup-real-e2e.sh` also need the `parkio-postgres-*` containers
running, take the test-account credentials from `PARKIO_REAL_*` env vars, and refuse to
touch `--target production` unless `PARKIO_CONFIRM_PRODUCTION=I_UNDERSTAND`. See
[`frontend/README.md`](../frontend/README.md) §"Real-stack E2E" for the full runbook.
